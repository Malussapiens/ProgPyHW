Архитектура приложения.
Модуль UI:
    Ввод данных от пользователя
        - приглашение
        - проверка правильности ввода
        - преобразование данных к формату записи
        - запись данных в файл
    Вывод данных на экран

Модуль работы с файлом:
    Загрузка (путь, имя файла):
        - проверяем есть ли файл, если есть, то:
            открытие на чтение
            закрытие файла
          иначе:
            сообщение об ошибке
            возвращаем None
        - считываем файл построчно в строку
        - возвращаем строку
    Сохранение (информация, путь, имя файла):
        - проверяем есть ли файл, если есть, то:
            открытие на дозапись
          иначе:
            создаем файл
            открытие на дозапись
        - закрытие файла

Модуль обработки данных
Формат записи (127 знаков):
Имя(12 знаков), Фамилия(12 знаков), отчество(15 знаков), номер телефона(15 цифр), примечание(73 знака)    
Формат файла:
    1) в файле на одной строке хранится одна часть записи, пустая строка - разделитель*
        *Фамилия_1*
        *Имя_1*
        *Телефон_1*
        *Описание_1*
        
        *Фамилия_2*
        *Имя_2*
        *Телефон_2*
        *Описание_2*
        
        *и т.д.
    2) на одной строке хранится все записи, символ разделитель - ','
    
        Фамилия_1,Имя_1,Телефон_1,Описание_1
        Фамилия_2,Имя_2,Телефон_2,Описание_2
        
        *и т.д.*
